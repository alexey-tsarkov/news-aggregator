# Техническое задание

## Стек технологий

* PHP >= 8.1
* Symfony
* Composer
* Git
* MySQL
* PHP-FPM
* Nginx
* RabbitMQ

## Описание задачи

Необходимо реализовать web-приложение новостного агрегатора с системой регистрации пользователей, импортом новостей из внешних источников, административной панелью и realtime-уведомлениями.

Верстка и визуальная составляющая не являются приоритетом. Основной акцент делается на backend-часть, архитектуру приложения и корректность работы функционала.

## Основной функционал

### 1. Регистрация и авторизация

Необходимо реализовать:
* регистрацию пользователей;
* авторизацию пользователей;
* подтверждение регистрации через Telegram или MAX.

Пароли пользователей должны храниться в хэшированном виде.

После регистрации пользователь не должен иметь возможности авторизоваться до подтверждения регистрации.

Код подтверждения должен иметь ограниченный срок действия.

Будет плюсом реализация:
* повторной отправки кода подтверждения;
* ограничения количества попыток ввода кода;
* ограничения частоты отправки кодов.

### 2. Telegram и RabbitMQ

Для подтверждения регистрации необходимо:
* создать Telegram/MAX-бота;
* создать Telegram/MAX-канал или группу;
* добавить бота в созданный канал или группу;
* реализовать отправку кодов подтверждения через Telegram/MAX.

Отправка сообщений должна выполняться исключительно асинхронно через RabbitMQ consumer.

При регистрации пользователя задача на отправку кода подтверждения должна помещаться в очередь RabbitMQ, после чего отдельный consumer должен выполнить отправку сообщения через Telegram/MAX-бота.

Необходимо сохранять информацию:
* о создании кода подтверждения;
* о постановке задачи в очередь;
* об успешной или неуспешной отправке сообщения.

Будет плюсом:
* обработка ошибок отправки;
* retry-механизм;
* логирование ошибок.

### 3. Импорт новостей

Необходимо реализовать импорт новостей минимум из одного внешнего источника:
* через API;
* либо через HTML-парсинг сайта.

Для каждой новости необходимо сохранять:
* заголовок;
* краткое описание;
* полный текст новости;
* дату публикации;
* источник получения;
* дату получения новости порталом.

Все новости должны храниться локально в базе данных.

Необходимо предусмотреть защиту от дублирования новостей.

Будет плюсом реализация универсального механизма подключения нескольких источников новостей.

### 4. Панель управления

Необходимо реализовать административную панель, в которой должны отображаться:

#### Пользователи

* успешно зарегистрированные пользователи;
* пользователи с незавершённой регистрацией;
* дата регистрации;
* статус подтверждения.

#### Коды подтверждения

* код подтверждения;
* пользователь;
* статус отправки;
* дата создания;
* дата отправки;
* дата подтверждения.

#### Новости

* список всех сохранённых новостей;
* источник новости;
* дата публикации;
* дата получения.

Список новостей должен поддерживать постраничную навигацию.

### 5. Поиск новостей

После авторизации пользователь должен видеть:
* поле поиска;
* 10 последних полученных новостей.

При вводе текста в поле поиска должен выполняться поиск по данным, находящимся в базе данных.

Отображение результатов поиска должно происходить без перезагрузки страницы посредством AJAX/fetch/XHR-запросов.

Будет плюсом:
* debounce поиска;
* полнотекстовый поиск;
* подсветка найденных совпадений.

### 6. Realtime-уведомления

Будет большим плюсом реализация realtime-уведомлений с использованием технологии COMET:
* SSE;
* либо WebSocket.

Все авторизованные пользователи, находящиеся на портале, должны получать визуальные уведомления о следующих событиях:
* регистрация нового пользователя;
* авторизация пользователя;
* получение новых новостей порталом.

Способ отображения уведомлений остаётся на усмотрение разработчика.

## Результат выполнения

Результатом выполнения задания должен быть:
* исходный код проекта;
* инструкция по настройке Telegram/MAX-бота;
* развернутый demo-stand на сервере.

*Важно: использование Docker/Docker Compose не допускается в качестве основного способа развертывания.*

Кандидат должен развернуть проект в классическом серверном окружении:
* Nginx + PHP-FPM + MySQL + RabbitMQ.

## Требования к серверу

Разработчик должен предоставить развернутый demo-stand на сервере.

Допускается использование:
* VPS;
* локальной виртуальной машины;
* выделенного тестового сервера.

Операционная система не принципиально.
